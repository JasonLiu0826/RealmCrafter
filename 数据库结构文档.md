### 12. ⚙️ 用户高阶配置表 (user_config)
**用途**: 存储极客用户（尤其是BYOK用户）的自定义AI参数

> ⚠️ **安全红线**: 用户的实际API密钥绝对不可存入此表，必须由客户端本地加密存储

| 字段名 | 数据类型 | 允许NULL | 键类型 | 默认值 | 说明 |
|--------|----------|----------|--------|--------|------|
| user_id | BIGINT | NO | PRI | NULL | 用户ID(主键，同时也是外键) |
| preferred_model | VARCHAR(64) | YES | | 'realm_crafter_v1' | 首选大模型 |
| chaos_level | DECIMAL(3,2) | NO | | 0.70 | 混沌阈值(对应LLM的温度0.1-1.0) |
| memory_depth | INT | NO | | 4000 | 记忆溯源深度(上下文最大Token数) |
| update_time | DATETIME | NO | | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**外键约束**: `fk_config_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE

## 🆕 新增表创建SQL

```sql
-- 创建用户高阶配置表
CREATE TABLE `user_config` (
  `user_id` BIGINT NOT NULL PRIMARY KEY COMMENT '用户ID(主键，同时也是外键)',
  `preferred_model` VARCHAR(64) DEFAULT 'realm_crafter_v1' COMMENT '首选大模型',
  `chaos_level` DECIMAL(3,2) NOT NULL DEFAULT '0.70' COMMENT '混沌阈值(对应LLM的温度0.1-1.0)',
  `memory_depth` INT NOT NULL DEFAULT 4000 COMMENT '记忆溯源深度(上下文最大Token数)',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT `fk_config_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户高阶配置表';

-- 创建广告变现流水表
CREATE TABLE `ad_view_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY,
  `user_id` BIGINT NOT NULL COMMENT '用户ID(外键)',
  `ad_platform` ENUM('ADMOB','PANGLE','UNITY') NOT NULL COMMENT '广告联盟来源',
  `ad_type` ENUM('REWARD_VIDEO','INTERSTITIAL') NOT NULL COMMENT '广告类型(激励视频/插屏)',
  `reward_granted` VARCHAR(64) NOT NULL COMMENT '发放的奖励（如：5000_TOKEN）',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY `idx_user_time` (`user_id`, `create_time`),
  CONSTRAINT `fk_ad_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='广告变现流水表';
```
